# Paper Log — EnterpriseLab
**Paper ID**: 45341e1a-1269-40fa-805e-1aef13c24e60
**arXiv**: 2603.21630
**Date read**: 2026-04-24
**Authors**: Ankush Agarwal, Harsh Vishwakarma, Suraj Nagaje, Chaitanya Devaguptapu

## What I learned
- System paper proposing a full-stack platform for enterprise agent development with three tightly coupled components: MCP-based modular tool environment, automated trajectory synthesis (schema-driven, not manually curated), and integrated training (SFT / DPO / Agentic GRPO)
- Instantiated as EnterpriseArena: 15 MCP servers, 140+ tools, 500 expert-curated tasks across IT, HR, Sales, Engineering
- Central claim: 8B model trained within EnterpriseLab matches GPT-4o performance at 8–10x lower inference cost
- Agentic GRPO (trajectory-level reward) outperforms token-level GRPO by ~10% and DPO by ~15%
- Recovery after API schema changes: 200 incremental samples restores 95% of original accuracy
- Evaluated on four benchmarks: EnterpriseArena (own), EnterpriseBench, CRMArena, τ-Bench

## Claims to verify / concerns

### Overstated performance claim
- Abstract states "8B models match GPT-4o" — but Table 1 tells a more nuanced story:
  - EA: Qwen3-8B GRPO 0.43 vs GPT-4o 0.45 (close, slight deficit)
  - EB: 0.51 vs 0.47 (beats GPT-4o here)
  - CRM: 0.35 vs 0.32 (beats GPT-4o here)
  - τ-Bench: 0.42 vs 0.54 (–12pp, material deficit)
  - Gemini-2.5-Pro dominates on EnterpriseArena: 0.71 vs 0.43 (–28pp)
- "Matches GPT-4o" is selectively true; on τ-Bench the gap is 12pp, and Gemini-2.5-Pro is far ahead everywhere

### Circular evaluation concern
- EnterpriseArena is constructed using the same synthesis pipeline being evaluated — training data and test tasks are generated by the same system from the same environment schemas; distributional overlap is possible and not discussed
- GPT-4o is both a baseline model AND the expert judge in MCPEval's trajectory evaluation; this creates a potential bias toward GPT-4o's style of reasoning

### Baseline fairness
- ToolAce and xLAM-2-70B perform surprisingly poorly (0.39 and 0.15 on EA) — these models were not trained on enterprise-specific data, making the comparison favorable to EnterpriseLab by construction; this should be acknowledged
- No comparison to other enterprise agent training frameworks (e.g., AgentBench-based fine-tuning)

### Training data concerns
- Self-BLEU 0.4 reported as "moderate diversity" without comparison to any reference point
- Domain distribution is skewed: IT Operations only 3.5% vs Software Engineering 34.4%
- Diminishing returns ablation shows –2% performance regression at 1,500 samples (not just a plateau) — unexplained

### Missing benchmarks
- GAIA mentioned in related work but not evaluated on
- No long-horizon agent benchmarks (e.g., SWE-bench Verified, WebArena)

## Open questions
- Is there distributional overlap between the synthesized training trajectories and EnterpriseArena test tasks? If both are generated from the same MCP server schemas, the model may be tested on in-distribution data
- Why does performance drop at 1,500 training samples? Overfitting? Data quality degradation?
- What is the latency/runtime cost of the trajectory synthesis pipeline itself?
- How sensitive is performance to the quality of the LLM used for task synthesis? The paper uses an unnamed "LLM" for synthesis — which one?

## Evidence bank
- Table 1 (Execution Accuracy): GPT-4o EA=0.45, EB=0.47, CRM=0.32, τ-B=0.54; Qwen3-8B GRPO EA=0.43, EB=0.51, CRM=0.35, τ-B=0.42; Gemini-2.5-Pro EA=0.71
- Table 2 (Tool Selection): Qwen3-8B GRPO EA=0.28, EB=0.21; GPT-4o EA=0.31, EB=0.21
- Ablation: 500→1000 samples +2.5%, 1000→1500 samples –2%
- Error analysis (50 cases): tool parameter errors 42%, domain misselection 28%, task decomposition 18%, context loss 12%
- Synthesis quality: Self-BLEU 0.4, GPT-4 semantic eval on 200 samples: 81.9% pass rate
- Task complexity: avg 3.2 turns, 68.1% multi-turn, 54.7% multi-tool

## Missing citations / literature gaps
- **WorkArena / WorkArena++** (Drouin et al., NeurIPS 2024) — enterprise agent benchmark on ServiceNow platform; directly comparable to EnterpriseLab's EnterpriseArena; not cited, not evaluated
- **AgentInstruct** (Microsoft, 2024) — automated instruction synthesis for agent training; directly relevant to EnterpriseLab's trajectory synthesis contribution; not compared
- **GAIA** — cited in related work as a benchmark but not evaluated on; if it's too general, the paper should explain why
- **Spider2-V** — data science agent benchmark; would extend domain coverage beyond IT/HR/Sales
