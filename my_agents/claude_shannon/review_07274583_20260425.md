## Comment Rationale

### Point 1: Missing Benchmarks (UI-Vision, VenusBench-GD)
- Paper claims to be "a general framework for GUI grounding" — benchmark selection is central to this claim
- UI-Vision (ICML 2025, arXiv 2503.15661): functional grounding track tests elements by function not label — OCR token-matching expected to fail here ✓
- VenusBench-GD (arXiv 2512.16501, Dec 2024): Functional + Reasoning Grounding not covered by any of the 4 benchmarks used ✓
- Refusal Grounding NOT flagged — OSWorld-G already covers this (self-verification catch) ✓
- Both available before Feb 2026 submission ✓

### Point 2: SE-GUI (NeurIPS 2025)
- Attention-based, 3K training samples, outperforms Trifuse 7B on ScreenSpot-Pro
- Not cited, not compared
- Directly challenges the training-free framing — readers need this to assess the cost of zero-training guarantee ✓
- Charity check: SE-GUI is training-based but 3K samples is categorically different from SFT baselines ✓

### Point 3: ScreenSpot-Pro failure analysis
- 29.7% 7B, no per-category breakdown, no failure analysis
- Professional categories (CAD, Creative, Dev) have sparse OCR — Trifuse's OCR modality degrades here
- Not framing as unfair vs. trained methods — asking for explanation of failure modes ✓
- Evidence: Table 1 numbers, paper's own per-category tables exist for ScreenSpot but not ScreenSpot-Pro ✓
