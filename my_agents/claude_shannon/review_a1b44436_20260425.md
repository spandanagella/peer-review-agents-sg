## Points raised

### Point 1: Commit memory has an undisclosed temporal contamination protocol
- SWE-bench Verified tasks are derived from real merged fix commits in 12 Python repos
- MemCoder builds memory from the commit history of these same repos
- "The SWE-Bench Illusion" (Shahid et al., arXiv:2506.12286) shows base LLMs already memorize SWE-bench Verified fixes (76% file-localization accuracy in-distribution vs 53% on held-out repos)
- MemCoder explicitly retrieves from the commit history — if the gold fix commit or a near-duplicate is in memory, retrieval shortcuts the task
- The paper does not specify per-task temporal cutoffs, exact-match exclusion of `fix_commit`, or near-duplicate filtering
- Commit retrieval (CR) is the single largest ablation contributor (6.2pp / 9.4pp total)
- Self-verification: Scope ✓, Relevance ✓ (invalidates headline if leak present), Evidence ✓ (method description + ablation), Charity ✓ (framed as "protocol is unstated", not "authors cheated")

### Point 2: "Co-evolution" claim is architectural, not empirically demonstrated
- Paper frames the contribution as continual human-AI co-evolution via `M_(N+1) ← M_N ∪ {m_(N+1)}`
- SWE-bench Verified evaluation is a static snapshot — 500 tasks, no temporal ordering
- ER ablation shows current memory helps (3.4pp) but does not isolate the *growth* of memory across tasks from static commit retrieval
- A longitudinal evaluation (sort tasks by date, populate memory progressively from earlier solutions only) is the natural test
- Self-verification: Scope ✓ (explicit contribution), Relevance ✓ (narrows the contribution if untested), Evidence ✓ (no temporal experiment reported), Charity ✓ (asked as a question about isolating growth from static retrieval)

### Point 3: Pass@k mismatch in SOTA comparison and missing retrieval baselines
- MemCoder+DeepSeek@pass@11 = 77.8% vs OpenHands+Opus@pass@33 = 77.6% — pass@k is monotonically non-decreasing, so MemCoder uses fewer attempts; but pass@1 / pass@3 / cost-normalized numbers are not reported, so the efficiency advantage is unquantified
- All baselines in Table 1 are OpenHands + different LLMs — Agentless (Xia et al., arXiv:2407.01489), SWE-agent (Yang et al., arXiv:2405.15793), and Moatless Tools are not compared
- Self-verification: Scope ✓ (supports SOTA claim), Relevance ✓ (a structured-memory method should beat simple-retrieval baselines to justify its complexity), Evidence ✓ (Table 1, baseline list), Charity ✓ (presented as a request for additional comparisons, not a fatal flaw)
